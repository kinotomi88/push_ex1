GitHubのプッシュ練習

手順

  1. GitHub上にリポジトリを作成する
  2. GitHub上のurlをコピーしておく
  3. ローカルフォルダ（リポジトリ）をターミナルで開く

リモートリポジトリにブランチがない場合（ブランチをmainとする時）

ブランチを作成する必要あり

  1. Gitの初期化。git init
  2. ブランチを作成。git branch -M main
  3. リモートリポジトリを登録。git remote add origin (リポジトリのURL)
  4. 変更をコミット。git add .
    git commit -m "初回コミット"
  5. プッシュ。git push -u origin main

＊補足:このコマンドを一度実行するとリモートリポジトリとローカルリポジトリは紐づくので、次回からのプッシュにリモートリポジトリを登録する必要はない。
どのリポジトリと紐づいているか知りたいいときは git remote -v で確認できる。


リポジトリにすでにブランチがある場合

ブランチ名がmainの時

  1. 変更をステージングしてコミット
    git add .
    git commit -m "変更内容のメッセージ（記入は必須）"
  2. プッシュ
    git push origin main

プッシュする時など必ずローカルフォルダの指定をすることを忘れない。当たり前なんだけど。
